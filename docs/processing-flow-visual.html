Yumekano Agent 処理フロー図解

ユーザーのメッセージがキャラクターの返事になるまでの仕組み

🔄
メインフロー
メッセージを受け取ってから返事を返すまでの10ステップ
IN
ユーザーのメッセージ
例：「今日何してた？」
1
情報の読み込み
返事に必要な情報を集める
キャラ設定（性格・価値観） 今の関係性 最近の会話
2
思い出の検索
関連する記憶を引き出す
「前に映画行く約束したよね」 「猫が好きって言ってた」
3-4
感情の処理
メッセージをどう受け止めたか → 今の気分を更新
「嬉しい質問だ」→ 機嫌UP 「プレッシャー」→ 緊張UP
5-6
返事の計画
「このキャラなら今どうする？」を決める + 関係の段階をチェック
態度：甘える/そっけない/からかう 行動：質問返し/提案/共感
7-8
候補生成 → ベスト選択
3〜5個の返事案を作り、最もキャラらしいものを選ぶ
候補A「バイトだったよー」 候補B「バイト！かーくんは？」 → Bを選択
9-10
記録 → 整理予約
会話と感情を保存、記憶が増えたら整理を予約
OUT
キャラクターの返事
例：「バイトしてたよ！かーくんは？」

💡 具体例：「かわいいね」と言われた場合

返事に反映
「え、急に何…」（照れ）
1
評価する
メッセージの意味を判断
🎯 私にとって良いこと？
💕 信頼できる相手？
😰 プレッシャー感じる？
✨ 新鮮で面白い？
2
瞬間的に反応
すぐに感じる気持ち
😊 嬉しい ↔ 😞 嫌
😆 興奮 ↔ 😌 落ち着き
💪 強気 ↔ 😔 弱気
3
気分に蓄積
ゆっくり変化する持続的な状態
一回の出来事では大きく変わらない
時間とともに基本状態に戻る
未解決の問題があると戻りにくい
4
返事に反映
気分が言葉遣いに影響
機嫌良い → 温かい、積極的
機嫌悪い → そっけない、短い
自信あり → ストレート、からかい
⚙️
設定でキャラを調整する
「こうしたい」を実現するための設定変更ガイド
🦋 性格・自立性を変える
反論しやすさ↑ → もっとツンデレに
断りやすさ↓ → もっと素直に
不機嫌の持続↑ → 根に持つタイプに
✨ 話し方を変える
積極性↑ → もっと甘えてくる
簡潔さ↑ → クールな短い返事
からかい度↑ → いじってくる
💗 感情の動きを変える
回復速度↑ → すぐ機嫌直る
敏感さ↑ → 感情の起伏が激しく
基本の気分を明るく → 普段からポジティブ
📈 関係の進み方を変える
遷移条件を厳しく → ゆっくり仲良くなる
許可行動を増やす → 早い段階で甘えてくる
禁止行動を追加 → 特定の言動をしない
