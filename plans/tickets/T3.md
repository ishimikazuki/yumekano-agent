## 1. Ticket ID

`T3`

## 2. Goal

Implement a deterministic integrator that maps CoE appraisal into PAD and pair-state deltas.

## 3. Why this ticket exists

Meaning extraction and state integration must stay separate so emotional state updates are stable, testable, and not tied to regex heuristics.

## 4. In scope

- Implement or complete the integrator that consumes relational appraisal axes.
- Include instantaneous effect, slow mood carry-over, decay, inertia, clamp, open-thread bias, and phase modifier behavior.
- Update PAD, `trust`, `affinity`, `conflict`, and `intimacyReadiness`.
- Keep a small deterministic abuse and consent guardrail layer only as override logic.
- Read weights from config or character emotion spec, not from workflow code.
- Add a feature flag for later old and new path comparison readiness.

## 5. Out of scope

- Production workflow wiring.
- Draft or sandbox persistence changes.
- Prompt bundle consolidation.
- Publish, versioning, or ranker gate work.
- Starting old and new trace comparison before `T4`.
- Later tickets that MUST NOT start yet: `T4`, `T5`, `T6`, `T7`, `T8`, `T9`.

## 6. Required code areas

- `src/lib/rules/pad.ts`
- Integrator code such as `src/lib/rules/coe-integrator.ts` or its replacement
- Character emotion config or spec read paths
- Integrator tests under `tests/`
- Feature flag definition path if one already exists

## 7. Forbidden code areas

- `src/mastra/workflows/chat-turn.ts`
- `src/mastra/workflows/draft-chat-turn.ts`
- Prompt storage and runtime loading files
- Publish, workspace, migration, and release flow files
- Ranker architecture files

## 8. Acceptance criteria

- The integrator does not depend on regex pattern matching.
- The example delta bands from the master plan are covered by tests for compliment, explicit insult, apology after hurt, and repeated pressure.
- Quiet turns decay back toward baseline smoothly.
- Open threads influence bias as part of the update.
- A feature flag exists for later comparison, but production comparison is not turned on in this ticket.

## 9. Required tests

- Delta-band tests for compliment
- Delta-band tests for explicit insult
- Delta-band tests for apology after hurt
- Multi-turn accumulation tests for repeated pressure
- Decay and inertia tests for quiet turns
- Open-thread bias tests

## 10. Stop condition

- Stop if the implementation starts depending on regex heuristics again.
- Stop if weights must be hardcoded into workflow code.
- Stop if enabling comparison would require `T4` production wiring.

## 11. Risks / notes

- The master plan requires config-driven weights but does not name one exact source file; use the existing emotion spec or config path rather than creating a second weight system.
- Guardrail logic must stay override-only in this ticket; do not let it become a second appraisal engine.
