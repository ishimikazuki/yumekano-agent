## 1. Ticket ID

`T2`

## 2. Goal

Implement the CoE Evidence Extractor so semantic interpretation returns structured evidence and relational appraisal output.

## 3. Why this ticket exists

Regex or pattern-based appraisal must be separated from meaning extraction before deterministic integration can be trusted.

## 4. In scope

- Add or complete a dedicated CoE or emotion-analysis agent module.
- Feed it the required context: user message, recent dialogue, current phase, pair state, working memory, retrieved facts, events, threads, observations, and open threads.
- Validate and retry malformed model output.
- Add mocked or fixture-based tests for insult, apology, pressure, and boundary-crossing cases.
- Keep CI independent from live model calls.

## 5. Out of scope

- PAD update logic.
- Direct pair-metric delta calculation.
- Production or draft workflow wiring.
- Prompt bundle unification beyond the extractor's own prompt storage.
- Later tickets that MUST NOT start yet: `T3`, `T4`, `T5`, `T6`, `T7`, `T8`, `T9`.

## 6. Required code areas

- `src/mastra/agents/` for the extractor module
- Existing CoE schema and contract files from `T1`
- Extractor prompt storage path if one already exists
- `tests/fixtures/`
- Extractor unit tests under `tests/`

## 7. Forbidden code areas

- `src/lib/rules/pad.ts`
- Integrator files such as `src/lib/rules/coe-integrator.ts`
- `src/mastra/workflows/chat-turn.ts`
- `src/mastra/workflows/draft-chat-turn.ts`
- Publish, versioning, or ranker gating files

## 8. Acceptance criteria

- Structured output is testable for insult, apology, repeated pressure, and boundary-crossing intimacy escalation.
- Parser failure has a safe fallback path.
- Unit tests pass without live model calls.
- The old heuristic path still exists.

## 9. Required tests

- Mocked extractor tests for the four required semantic cases
- Validation and retry tests for malformed outputs
- Safe fallback tests when validation still fails
- CI-safe tests that run without external model access

## 10. Stop condition

- Stop if the extractor cannot be tested without live model calls.
- Stop if the change starts to define PAD math or pair-metric integration.
- Stop if the work starts to wire `T3+` behavior into production or draft paths.

## 11. Risks / notes

- The master plan mentions `src/lib/prompts/` or canonical prompt storage, but this repository may already store prompts elsewhere; reuse the existing prompt source rather than creating a new one.
- Keep any uncertainty in extractor output as explicit `uncertaintyNotes`; do not hide ambiguity in downstream logic.
