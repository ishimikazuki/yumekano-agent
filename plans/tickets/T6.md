## 1. Ticket ID

`T6`

## 2. Goal

Consolidate prompt storage, loading, editing, and runtime use into one canonical prompt bundle shape.

## 3. Why this ticket exists

Prompt changes are unsafe while checked-in defaults, persisted prompts, draft prompts, and runtime loading disagree about what the prompt bundle actually is.

## 4. In scope

- Define one canonical prompt bundle shape.
- Include planner, generator, `generatorIntimacy`, ranker, and CoE appraiser prompts in that shape.
- Align checked-in default prompts, persisted prompts, draft prompts, and runtime loading to the same shape.
- Remove divergent inline prompts or mark them deprecated explicitly.
- Add round-trip tests.
- Add migrations or compatibility layers only where existing data would otherwise break.

## 5. Out of scope

- Prompt tuning or wording optimization.
- Production or sandbox state logic changes unrelated to prompt shape.
- Publish and versioning flow consolidation.
- Ranker gate work, rollout, or legacy cleanup.
- Later tickets that MUST NOT start yet: `T7`, `T8`, `T9`.

## 6. Required code areas

- `src/lib/schemas/prompts.ts`
- Prompt repository paths
- Workspace repository paths involved in draft editing
- Seed or default prompt definitions
- Runtime prompt loading path
- Draft prompt editing path
- Prompt persistence tests under `tests/`

## 7. Forbidden code areas

- Emotional runtime logic and integrator math files
- Publish and release flow files except compatibility touchpoints strictly required by prompt persistence
- Ranker gate and full eval files
- Legacy heuristic cleanup files

## 8. Acceptance criteria

- The canonical prompt bundle can be saved and loaded without shape loss.
- `generatorIntimacy` and CoE appraiser prompts persist reliably.
- Runtime uses only the canonical bundle shape.
- A compatibility path exists for old data.

## 9. Required tests

- Prompt round-trip persistence tests
- Compatibility tests for existing prompt data
- Runtime loading tests proving the canonical bundle is used
- Draft editing tests for the canonical prompt bundle shape

## 10. Stop condition

- Stop if two prompt source-of-truths would remain after the change.
- Stop if existing data cannot be migrated or adapted without making up new prompt kinds.
- Stop if the work starts to change publish and versioning flow beyond prompt compatibility.

## 11. Risks / notes

- The exact existing names or paths for `generatorIntimacy` and CoE appraiser prompts may differ in code; map them explicitly and deprecate aliases rather than silently inventing additional prompt categories.
- Keep this ticket structural. Do not use it to tune prompt content quality.

## 12. Handoff prompt template

```text
Work only on T6 using plans/tickets/T6.md and COE_PAD_REPAIR_PLAN.md.
Unify prompt storage and runtime shape with failing round-trip tests first, keep the change structural rather than tuning-oriented, and stop if the work expands into publish, ranker, or rollout changes.
```
