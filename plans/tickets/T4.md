## 1. Ticket ID

`T4`

## 2. Goal

Wire the new CoE to integrator pipeline into the production `chat_turn` path.

## 3. Why this ticket exists

The new contracts and integrator are not useful until production uses them, persists the resulting state, and records enough trace data to compare old and new behavior safely.

## 4. In scope

- Replace the direct heuristic appraisal call in production with the new CoE and integrator path.
- Persist updated pair metrics as well as PAD.
- Pass evidence, relational appraisal, current state, state delta, and retrieved observations into planner, generator, and ranker context.
- Extend trace persistence with CoE evidence, relational appraisal, state deltas, and optional old and new comparison data when the feature flag is on.
- Keep the change limited to the production path.

## 5. Out of scope

- Draft or sandbox persistence and carry-over work.
- Prompt source-of-truth cleanup.
- Publish, versioning, ranker gate work, or legacy cleanup.
- Later tickets that MUST NOT start yet: `T5`, `T6`, `T7`, `T8`, `T9`.

## 6. Required code areas

- `src/mastra/workflows/chat-turn.ts`
- The production execution path if shared code lives in `src/mastra/workflows/execute-turn.ts`
- Planner, generator, and ranker context injection paths
- Trace persistence paths
- Production integration tests under `tests/`

## 7. Forbidden code areas

- `src/mastra/workflows/draft-chat-turn.ts`
- Sandbox persistence repositories and migrations except for read-only dependency alignment
- Prompt schema and prompt storage paths
- Publish and versioning flow files
- Ranker gate logic beyond consuming the new context

## 8. Acceptance criteria

- A one-turn production integration test passes.
- A three-turn progression test passes.
- `trust`, `affinity`, `conflict`, and `intimacyReadiness` are persisted in production.
- The trace shows old and new differences when the feature flag is enabled.

## 9. Required tests

- One-turn production integration test
- Three-turn progression test
- Persistence test for PAD and pair metrics
- Trace test covering CoE evidence, appraisal, state delta, and old and new comparison data

## 10. Stop condition

- Stop if making production pass requires sandbox state changes; that belongs to `T5`.
- Stop if the work starts to consolidate prompt storage or publish flow.
- Stop if the change would add deterministic ranker gates; that belongs to `T8`.

## 11. Risks / notes

- The repository may route `chat_turn` through shared execution code; keep the functional scope production-only even if the implementation path is shared.
- Do not let trace expansion become a general trace redesign outside the new CoE and integrator payloads.
